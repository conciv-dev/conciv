## New npm package: `@conciv/extension-ios`

1. **`packages/publish/src/guards.ts` — add to `PUBLIC_PACKAGES`** (line 18 list). Without this,
   `assertPublicSet` aborts the release on drift. Add `'@conciv/extension-ios'`. Note the existing set
   uses the flattened names `@conciv/extension-terminal` / `@conciv/extension-whiteboard` etc. for the
   `packages/extensions/*` dirs — match that convention (`@conciv/extension-ios`).
2. **`.fallowrc.json` `publicPackages` — add ONLY if fallow actually flags it (M14).** This list is

   `fallow audit --changed-since main`; only
   if it flags `@conciv/extension-ios` exports as unused (and `fallow dead-code --trace 'file.ts:Symbol'`
   confirms it is genuinely a public-API entry fallow cannot reach, not a missing wiring) add it. Do **not**
   blanket-add "to follow terminal" — terminal is not there. `PUBLIC_PACKAGES` (step 1) is still required.
3. **package.json manifest fields** (guards check these for every public package):
   - `"name": "@conciv/extension-ios"`, `"homepage": "https://conciv.dev"`,
     `"repository": {"type":"git","url":"git+https://github.com/conciv-dev/conciv.git","directory":"packages/extensions/ios"}`,

     `"publishConfig": {"access": "public"}`, `"files": ["dist"]`, `"type": "module"`.
   - `exports`: `"."` → `dist/server.js` (server tools, the node-side entry), `"./client"` →

   - Description must note "Internal to `@conciv/it` (installed automatically, do not install directly)"

4. **Version:** all `@conciv/*` are `fixed` in `.changeset/config.json` (`fixed: [["@conciv/*"]]`), so the

## Native SDK — HYBRID: monorepo source, mirror-repo distribution (D2)

The Swift SDK is **not** an npm package and is **not** part of `@conciv/*` versioning. Source of truth is
in the monorepo (`native/swift/ConcivWidget/`, nested manifest — `04`); distribution is a **mirror repo**
that release CI publishes to. **The monorepo gains no root `Package.swift` and no bare-semver tags.**

- **Consumption URL = the mirror only.** Consumers pin
  `.package(url: "https://github.com/conciv-dev/conciv-swift.git", from: "1.0.0")` and reference
  `.product(name: "ConcivWidget", package: "conciv-swift")`. The monorepo URL is never a Swift consumption
  URL, so there is no one-way door (M-A9 resolved) — the mirror is a disposable, regenerable artifact.
- **Bare-semver tags live in the MIRROR repo.** SwiftPM's `from:`/`exact:` resolve semantic-version tags;
  the mirror tags with bare semver (`1.0.0`/`v1.0.0`). Because the tags live in a **separate** repo there is
  no collision with the monorepo's scoped npm tags at all — the earlier "all 285 tags scoped" collision
  argument is **moot** (and was slightly off: the monorepo has two non-semver tags too). No bare-semver tag
  is ever cut on the monorepo.
- **Mirror publish flow (sketch — wire into the release workflow, `.github/workflows/release.yml`).** On a
  release that touches the Swift source or the bridge, a publish step regenerates the mirror from the
  monorepo `native/swift/ConcivWidget/` tree:
  1. Ensure the in-repo fixture copy is synced (`pnpm --filter @conciv/extension-ios gen:fixtures`) and the
     `git diff --exit-code` gate is clean (below).
  2. Copy `native/swift/ConcivWidget/` (Sources + Tests + the nested `Package.swift`, promoted to the mirror
     **root**) into the mirror working tree — a `git subtree push --prefix native/swift/ConcivWidget` to the
     mirror, or a small publish script that rsyncs the tree and force-pushes a release branch. Nobody edits
     the mirror by hand; it is fully derived.
  3. Tag the mirror with the bare-semver version and push the tag. The mirror's committed fixtures copy is
     the same tree the in-repo gate already verified, so the published SDK is contract-consistent by
     construction.
- **Versioning policy:** the mirror's tag tracks **bridge protocol** compatibility, not the npm version.
  Bump when `BRIDGE_MAX_VERSION`, the wire, or the public `attach` API changes. The mirror README maps the
  Swift `x.y.z` ↔ minimum `@conciv/extension-ios` npm range (independent cadences).

## Bridge fixtures — hand-maintained table + in-repo drift-check (D3 / M12 / D2)

The bridge fixtures are a **hand-maintained example table** (`02` M11, D3 — NOT generated by walking
arbitrary zod schemas) and are **copied** into the Swift test tree; the copy may not drift (it gates
`swift test`, `07`):

- The canonical table lives at `packages/extensions/ios/fixtures/bridge/`; `pnpm --filter
@conciv/extension-ios gen:fixtures` writes the `*.json` from the hand-authored table and copies them to
  `native/swift/ConcivWidget/Tests/ConcivWidgetTests/Fixtures/bridge/` (an **in-repo** path copy).
- A CI step runs the command then `git diff --exit-code`, so an out-of-sync fixture fails the build —
  **in-repo, no cross-repo fetch** (D2 hybrid). The mirror publish (above) carries the already-synced copy.
