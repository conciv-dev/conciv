Pass options to the plugin. Every field is optional.

```ts title="vite.config.ts"
import mandarax from 'mandarax/plugin/vite'

mandarax({
  harness: 'claude',
  enabled: process.env.NODE_ENV !== 'production',
})
```

## Options

<TypeTable
  type={{
    enabled: {type: 'boolean', default: 'true', description: 'Mount the agent. Gate it on dev mode in real apps.'},
    harness: {type: 'string', default: '"claude"', description: 'Harness adapter id, such as claude or codex.'},
    harnessBin: {type: 'string', default: 'adapter default', description: 'Override the harness binary on PATH.'},
    testRunner: {type: 'string', default: '"vitest"', description: 'Test runner adapter id.'},
    widgetUrl: {
      type: 'string',
      default: 'MANDARAX_WIDGET_URL',
      description: 'Script src for the injected widget. Omit to skip injection.',
    },
    systemPrompt: {type: 'string', default: 'built-in', description: 'Appended to each agent turn.'},
    previewId: {type: 'string', default: '"local"', description: 'Correlates a resumable session.'},
    sessionId: {type: 'string', default: 'none', description: 'Resume an existing thread.'},
    port: {type: 'number', default: 'auto', description: 'Fixed engine port. Used by the Next.js integration.'},
    stateRoot: {type: 'string', default: 'process.cwd()', description: 'Root holding the .mandarax state directory.'},
    widget: {
      type: 'object',
      default: 'both layouts on',

    },
  }}
/>

<Callout>`claudePath` and `claudeSessionId` are deprecated. Use `harnessBin` and `sessionId`.</Callout>

## Common tasks

### Gate to dev only

```ts
mandarax({enabled: process.env.NODE_ENV !== 'production'})
```

### Switch harness

```ts
mandarax({harness: 'codex'})
```

See [Harnesses](/docs/harnesses) for the full list.

### Widget layouts

Two layouts ship, both on by default: the corner `modal` and the top drop-down `quickTerminal`.
Configure or disable each under `widget`.

```ts
mandarax({widget: {modal: {position: 'top-left'}}}) // move the corner button
mandarax({widget: {quickTerminal: {hotkey: ['Mod+`', 'Control+k']}}}) // set the hotkey(s)
mandarax({widget: {quickTerminal: false}}) // corner modal only
mandarax({widget: {modal: false}}) // quick terminal only
```

`modal.position` is one of `top-left`, `top-right`, `middle-left`, `middle-right`, `bottom-left`,
`bottom-right` (default), and the button is also draggable. See the
[Quick Terminal](/docs/usage/quick-terminal) guide for the drop-down layout.

### Override the binary

```ts
mandarax({harnessBin: '/usr/local/bin/claude'})
```

### Custom system prompt

```ts
mandarax({systemPrompt: 'Prefer Tailwind classes over inline styles.'})
```

### widgetUrl

On webpack and Rspack the plugin does not inject the widget. Set `widgetUrl` to the served widget
bundle so the script lands on your page.

```ts
mandarax({widgetUrl: '/mandarax-widget.js'})
```
