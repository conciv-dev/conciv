# Grounded live driving on conciv.dev (issue #58 track B, follow-up)

Fixes the demo gap found on first live verify of the connect flow: the widget pairs and chats,
but the agent is blind. Grab context is minified fiber noise, `data-conciv-source` is absent from
the prod build, and the connect workspace is an empty temp dir. The agent cannot ground anything
it sees and cannot show it understood the page.

Status: designed, approved 2026-07-15. Builds on
`2026-07-15-site-live-widget-connect-design.md` (shipped on branch `live-widget-connect`).

## Goal

A visitor grabs any element on conciv.dev and their agent resolves it to the real source file,
reads it, makes live page changes that reference it, and can show the real diff it would make.

## Gap mechanics (verified in code)

- `packages/page/src/react-bridge.ts` `locate()` prefers `closest('[data-conciv-source]')` and
  falls back to React fiber debug stacks. Minified prod builds make the fallback useless.
- The stamper exists as a pure transform: `addSourceToJsx` in
  `packages/extension-compiler/src/inject-source.ts` (`rel:line:col` per JSX opening element).
  The conciv plugin only applies it in dev (`apply: 'serve'`), so the deployed site is bare.
- `runConnect` seeds a `conciv-connect-` temp workspace with nothing. Perfect paths would
  resolve to no files.

## Design

Three pieces, all site- and connect-local. No changes to core, page tools, or the widget.

### 1. Annotations in the deployed site

A small vite plugin local to `apps/site` (in `vite.config.ts`, `apply: 'build'`) runs
`addSourceToJsx` over the site's own `src/**/*.tsx`. The deployed DOM then carries
`data-conciv-source="src/components/landing/hero.tsx:16:9"` on every element, and grab grounds
instantly through the existing attr path.

- `@conciv/extension-compiler` becomes a site devDependency.
- Paths are relative to `apps/site`, matching the workspace layout the connector creates.
- Cost: attribute bytes on a marketing page. Paths are public anyway (open-source repo).
- Site-local on purpose. Shipping annotations as a general `@conciv/it` prod feature waits for a
  second consumer.

### 2. Source manifest asset

The site build packs `apps/site/src/**` plus `package.json` into a JSON manifest
(`{"src/components/landing/hero.tsx": "<content>", ...}`) served as a static asset next to the
widget bundle, e.g. `/site-source.json`. Built at deploy, so it exactly matches the deployed
page. On pair, the connector downloads `<origin>/site-source.json` and writes the files into the
throwaway workspace with plain `fs`. No git, no tar, no new npm deps on either side.

- Generated by a sibling script (`scripts/build-source-manifest.mjs`) wired into the site
  `build` script next to the existing widget-bundle copy step.
- The asset is gitignored like the widget bundle.
- Text files only (tsx, ts, css, md, json). Binary assets are excluded.
- The connector gains a `seedWorkspace(origin, root)` step in `@conciv/connect`, running before
  `start()`. Failure to fetch the manifest degrades gracefully: log a warning, continue
  unseeded (the old behavior).

### 3. Workspace grounding for the agent

`seedWorkspace` also writes an `AGENTS.md` at the workspace root:

- This workspace contains the source of the page you are connected to (the conciv.dev landing
  page).
- `data-conciv-source` attributes on grabbed elements map to these files.
- File edits here are a local sandbox. Nothing redeploys. Use page tools for live changes, and
  show diffs when the user asks you to change something.

This fixes grounding at the source instead of prompt-patching the harness.

## Flow after the fix

1. Visitor pairs, connector seeds the workspace from `/site-source.json`, core starts.
2. Visitor grabs the hero heading. Grab reads `data-conciv-source` and returns
   `src/components/landing/hero.tsx:16`.
3. Agent reads the real file from its cwd, explains what it found, restyles live via page tools,
   and can produce the real diff.

## Testing

- Unit (site): manifest builder emits expected paths and skips binaries; annotation plugin
  stamps `data-conciv-source` (assert on the plugin's transform output for a fixture tsx).
- IT (`@conciv/connect`): `seedWorkspace` against a local static server writes files +
  `AGENTS.md` into the workspace; manifest fetch failure leaves the workspace usable.
- E2E (site, extend `live-connect.it.test.ts`): after connect, a grabbed element resolves to a
  workspace path that exists (fake harness; assert via the page/grab surface or the DOM
  attribute + workspace file check).

## Not doing (YAGNI)

- General prod-annotation feature in `@conciv/it`.
- `?core=` power-user override, page-reset chip.
- HMR or redeploy of visitor edits.
- Binary assets in the manifest.
