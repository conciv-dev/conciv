# Test-Runner Seam (`@opendui/aidx-runner`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **HARD RULES (see the spec's "Coding conventions" — non-negotiable):** No casts (`as` except `as const`), no `!`, no IIFEs, no `index.ts`. **Zod for all parsed data** — the runner child↔manager NDJSON uses `ChildMessageSchema` (`TestEventSchema` from protocol + list/error); validate via `Schema.safeParse(JSON.parse(line))`, NEVER hand-roll `isRecord`. `TestEvent`/`TestRunResult` types are inferred from Zod in `@opendui/aidx-protocol`. Every runner authored via `defineRunner`. Adapters in subfolders (`runner/<id>/<id>.ts`), no `index.ts`; shared child-driver is the only `_`-prefixed exception. Terse comments. Commit to `main`. **Verify each runner's reporter API (jest/node:test/playwright) against real online docs.**

> **REALIZED CORE LAYOUT (Plan 1, supersedes any flat name below).** All HTTP routes live under
> `@opendui/aidx-core/src/api/<domain>/`, paths `/api/<domain>/…`, entry named after its folder. The test
> route is `api/test-runner/test-runner.ts`, wired by **`registerTestRunnerRoutes(app, mgr)`** (mutates an h3 `H3` app;
> there is no `test-route.ts` / `makeTestRoute` factory and no `@opendui/aidx-core/test-route` subpath —
> route ITs build an `H3` app, call `registerTestRunnerRoutes`, and serve via srvx, exactly like Plan 1's
> `test/api/test-runner/test-runner.it.test.ts`). The vitest runner lives under `src/runner/vitest/{adapter,manager,child}.ts`
> with a `runner/registry.ts`. **Where this plan says `test-route.ts`/`makeTestRoute`, read
> `api/test-runner/test-runner.ts` + `registerTestRunnerRoutes(app, mgr)`.**

**Goal:** Extract the vitest-specific test-runner machinery out of `@opendui/aidx-vite-plugin` into a dedicated, runner-agnostic `@opendui/aidx-runner` package, behind the `TestRunnerManager` contract defined in `@opendui/aidx-protocol`. Ship four adapters — **vitest** (ported verbatim), **jest**, **node:test**, and **playwright** — each emitting the same `TestEvent` wrapper types across a clean-env fd3/NDJSON channel. Generalize `@opendui/aidx-core`'s test route to resolve a runner by id, retype the widget card over `TestEvent`/`TestRunResult` with no "vitest" string in it, and rename the CLI subcommand `tools vitest` → `tools test` (keeping `vitest` as a back-compat alias for one cycle).

**Architecture:** This is **Seam 2 — Test-runner adapters** from `docs/superpowers/specs/2026-06-13-aidx-adapter-arch-design.md`. The reusable core of today's `vitest-manager.ts` (clean-child spawn that strips `NODE_OPTIONS` + `VIBE_*`, an NDJSON-on-fd3 reader, and the cache/runBuffer/subscribe logic) becomes a shared `driver.ts` that returns a `TestRunnerManager`. Each runner adapter supplies only its child-script URL, its arg mapping, and its `TestRunnerCapabilities`; the child script does the dirty translation from the runner's native output into `TestEvent` NDJSON. Consumers (`@opendui/aidx-core` test route, `@opendui/aidx-widget` test card) speak `TestEvent`/`TestRunResult` only — zero runner imports. A `registry.ts` (`registerRunner` / `getRunner(id)` / `listRunners`) lets `@opendui/aidx-core` resolve `config.testRunner ?? 'vitest'` and lets third parties register adapters at runtime against the published `@opendui/aidx-protocol` interfaces.

**Tech Stack:** ESM-only, Node 22+, TypeScript 6 with `verbatimModuleSyntax` + `isolatedModules` + `noUncheckedIndexedAccess` (from `tsconfig.base.json`). Build via `tsdown` (multi-entry: each `child.ts` is its own output, spawned via `new URL(...)`). Tests via `vitest` (run from the package). Functions, never classes; no IIFEs; no `index.ts` barrels — each file is named after its contents and package `exports` subpaths point at named files. pnpm workspace + turbo. Each runner's native lib (vitest/jest/node:test/playwright) is the **previewed app's** dependency, resolved at runtime from its cwd via `createRequire` — never a static dependency of `@opendui/aidx-runner`.

**Assumes Plan 1 (foundation) complete.** Plan 1 has already:

- Renamed `@opendui/aidx-protocol/vitest-types` → `@opendui/aidx-protocol/test-types`, with `VitestEvent → TestEvent` and `RunResult → TestRunResult` (other type names — `TestState`, `TestError`, `Summary`, `FileState`, `TestRow`, `TestCaseLike`, `parseFailure` — unchanged).
- Defined `@opendui/aidx-protocol/runner-types` exporting the `TestRunnerManager` interface (`list` / `run` / `status` / `subscribeRaw` / `emitSnapshot` / `openUiServer` / `stop`), `RunArgs`, `ListResult`, `UiServerInfo`, and `TestRunnerCapabilities` (`{watch, uiServer, filterByName, failedOnly}`), plus a `TestRunner` adapter shape `{id, capabilities, childUrl, buildRunArgs, buildListArgs}`, **and the `defineRunner<T extends TestRunner>(adapter: T): T` typed factory** (the per-seam `define*` helper from the design's "Authoring adapters" section — locks the literal type, applies defaults, dev-asserts capability/method consistency). If Plan 1 named the adapter shape `TestRunnerAdapter` rather than `TestRunner`, `defineRunner`'s constraint uses that name; this plan imports `defineRunner` regardless.
- Created `@opendui/aidx-core` (the h3 engine) with vitest wired **inline** via a runner registry, and renamed `vitest-route.ts → test-route.ts`.

This plan **extracts** the runners into `@opendui/aidx-runner` and adds jest/node-test/playwright. Where Plan 1's exact `@opendui/aidx-protocol` shapes are referenced below, they are restated so each task is self-contained; if a name drifted in Plan 1, prefer the actual exported name and keep the rest of the wiring identical.

- [ ] Write `packages/runner/test/helpers.ts` — a generic tsx spawn-seam factory (parameterized by child URL, replacing the old vitest-only helper), plus an error spawn and a fake-child spawn:

```ts
import {spawn, type ChildProcess} from 'node:child_process'
import {createRequire} from 'node:module'
import {fileURLToPath, pathToFileURL} from 'node:url'
import type {SpawnRunner} from '../src/driver.js'

// ITs run the REAL adapter child via tsx (no build step) so they exercise the true
// child-process path. tsxSpawnFor(childTsUrl) returns a SpawnRunner bound to that child.
const require = createRequire(import.meta.url)
const tsxEntry = pathToFileURL(require.resolve('tsx')).href

export function tsxSpawnFor(childTsUrl: URL): SpawnRunner {
  const childTs = fileURLToPath(childTsUrl)
  return (args, cwd) => {
    const env: NodeJS.ProcessEnv = {...process.env}
    delete env.NODE_OPTIONS
    return spawn(process.execPath, ['--import', tsxEntry, childTs, ...args], {
      cwd,
      env,
      stdio: ['ignore', 'pipe', 'pipe', 'pipe'],
    })
  }
}

// Spawn seam whose child immediately emits the out-of-process failure signal (an `error`
// NDJSON message on fd 3), so the driver surfaces a typed RunnerUnavailableError.
export function errorSpawnRunner(reason: string): SpawnRunner {
  const payload = JSON.stringify({type: 'error', reason})
  return () =>
    spawn(process.execPath, ['-e', "require('fs').writeSync(3, process.argv[1] + '\\n')", payload], {
      stdio: ['ignore', 'pipe', 'pipe', 'pipe'],
    })
}
```

```ts
import {spawn, type ChildProcess} from 'node:child_process'
import {createInterface} from 'node:readline'
import {fileURLToPath} from 'node:url'
import type {Readable, Writable} from 'node:stream'
import type {FileState, Summary, TestEvent, TestRunResult} from '@opendui/aidx-protocol/test-types'
import type {
  ListResult,
  RunArgs,
  TestRunner,
  TestRunnerManager,
  UiServerInfo,
} from '@opendui/aidx-protocol/runner-types'

// Drives a previewed app's test runner OUT OF PROCESS. The aidx dev server runs under an
// import-in-the-middle preload; embedding the runner in that same process corrupts the live
// app's transforms. So each request spawns the adapter's child.js as a CLEAN child
// (NODE_OPTIONS + VIBE_* stripped) that streams TestEvents as NDJSON on fd 3. This module is
// the shared, runner-agnostic core of the old vitest-manager.ts: the adapter supplies only
// the child URL + arg mapping; the driver owns spawn/read/cache/subscribe.

export type SpawnRunner = (args: string[], cwd: string) => ChildProcess
export type MakeManagerOptions = {spawnRunner?: SpawnRunner}

// Control messages a child may emit alongside TestEvents.
type ListMessage = {type: 'list'; files: ListResult['files']}
type ErrorMessage = {type: 'error'; reason: string}
type ChildMessage = TestEvent | ListMessage | ErrorMessage

const RUNNER_UNAVAILABLE_TAG = 'aidx:runner-unavailable'
export type RunnerUnavailableError = Error & {readonly [RUNNER_UNAVAILABLE_TAG]: true; readonly available: false}

export function runnerUnavailableError(reason: string): RunnerUnavailableError {
  // Build the tagged error by extending Error, not by asserting a plain Error into the brand.
  return Object.assign(new Error(`runner unavailable: ${reason}`), {
    [RUNNER_UNAVAILABLE_TAG]: true as const,
    available: false as const,
  })
}
function isRecord(v: unknown): v is Record<string, unknown> {
  return typeof v === 'object' && v !== null
}
export function isRunnerUnavailable(e: unknown): e is RunnerUnavailableError {
  return e instanceof Error && isRecord(e) && e[RUNNER_UNAVAILABLE_TAG] === true
}

// Discriminated-union guards over ChildMessage["type"] — no Extract<…> filters, no `as`.
// Typed as ReadonlySet<string> so `.has(msg.type)` accepts the wider ChildMessage type union
// without an assertion; membership is the runtime narrowing.
const TEST_EVENT_KINDS: ReadonlySet<string> = new Set<TestEvent['type']>([
  'snapshot',
  'run-start',
  'test',
  'file-end',
  'run-end',
])
function isTestEvent(msg: ChildMessage): msg is TestEvent {
  return TEST_EVENT_KINDS.has(msg.type)
}
function isListMessage(msg: ChildMessage): msg is ListMessage {
  return msg.type === 'list'
}
function isErrorMessage(msg: ChildMessage): msg is ErrorMessage {
  return msg.type === 'error'
}
function isRunEnd(e: TestEvent): e is Extract<TestEvent, {type: 'run-end'}> {
  return e.type === 'run-end'
}

const EMPTY_SUMMARY: Summary = {passed: 0, failed: 0, skipped: 0, durationMs: 0}

// Strip the dev server's IITM preload + every VIBE_* so the runner is an ordinary process.
function cleanEnv(base: NodeJS.ProcessEnv): NodeJS.ProcessEnv {
  const env: NodeJS.ProcessEnv = {...base}
  delete env.NODE_OPTIONS
  for (const key of Object.keys(env)) {
    if (key.startsWith('VIBE_')) delete env[key]
  }
  return env
}

// Narrow a child's extra stdio slot (typed Readable | Writable | null | undefined) to the
// readable fd-3 event channel via a type-predicate guard — never `child.stdio[3] as Readable`.
// A Readable exposes `.read()`; a Writable does not, so that method is the discriminator.
function isReadable(stream: Readable | Writable | null | undefined): stream is Readable {
  return stream !== null && stream !== undefined && 'read' in stream && typeof stream.read === 'function'
}
function readableChannel(stream: Readable | Writable | null | undefined): Readable | null {
  return isReadable(stream) ? stream : null
}

function defaultSpawnRunner(childUrl: URL): SpawnRunner {
  const childScript = fileURLToPath(childUrl)
  return (args, cwd) =>
    spawn(process.execPath, [childScript, ...args], {
      cwd,
      env: cleanEnv(process.env),
      stdio: ['ignore', 'pipe', 'pipe', 'pipe'],
    })
}

// Parse untrusted child output to `unknown`, then narrow with a guard over the discriminated
// `type` field — never `JSON.parse(line) as ChildMessage`.
const CHILD_MESSAGE_KINDS: ReadonlySet<string> = new Set([
  'snapshot',
  'run-start',
  'test',
  'file-end',
  'run-end',
  'list',
  'error',
])
function isChildMessage(value: unknown): value is ChildMessage {
  return isRecord(value) && typeof value['type'] === 'string' && CHILD_MESSAGE_KINDS.has(value['type'])
}
function parseMessage(line: string): ChildMessage | null {
  let value: unknown
  try {
    value = JSON.parse(line)
  } catch {
    return null
  }
  return isChildMessage(value) ? value : null
}

export function makeManager(adapter: TestRunner, cwd: string, options: MakeManagerOptions = {}): TestRunnerManager {
  const spawnRunner = options.spawnRunner ?? defaultSpawnRunner(adapter.childUrl)
  const subscribers = new Set<(e: TestEvent) => void>()
  // Annotate the fields directly so the literal is checked against the type — no `as` needed.
  const emptyResult: TestRunResult = {summary: EMPTY_SUMMARY, failures: [], tests: []}
  const cache: {last: TestRunResult; files: Map<string, FileState>} = {
    last: emptyResult,
    files: new Map<string, FileState>(),
  }
  const runBuffer: TestEvent[] = []
  const lifecycle: {child: ChildProcess | null} = {child: null}

  function emit(e: TestEvent): void {
    for (const cb of subscribers) cb(e)
  }
  function updateCache(e: TestEvent): void {
    if (e.type === 'run-start') {
      cache.files.clear()
      runBuffer.length = 0
    }
    if (e.type === 'file-end')
      cache.files.set(e.file, {file: e.file, state: e.ok ? 'pass' : 'fail', durationMs: e.durationMs})
    if (e.type === 'run-end') cache.last = {summary: e.summary, failures: e.failures, tests: e.tests}
    runBuffer.push(e)
  }

  function driveChild(args: string[], forward: boolean): Promise<ChildMessage[]> {
    return new Promise((resolve, reject) => {
      const child = spawnRunner(args, cwd)
      lifecycle.child = child
      // child.stdio[3] is typed `Readable | Writable | null | undefined`; narrow to the
      // readable fd-3 channel with a guard instead of asserting it.
      const channel = readableChannel(child.stdio[3])
      if (!channel) {
        reject(runnerUnavailableError('runner did not expose its event channel (fd 3)'))
        return
      }
      const messages: ChildMessage[] = []
      const errState: {reason: string | null} = {reason: null}
      const rl = createInterface({input: channel})
      rl.on('line', (line) => {
        const trimmed = line.trim()
        if (!trimmed) return
        const msg = parseMessage(trimmed)
        if (!msg) return
        if (isErrorMessage(msg)) {
          errState.reason = msg.reason
          return
        }
        messages.push(msg)
        if (forward && isTestEvent(msg)) {
          updateCache(msg)
          emit(msg)
        }
      })
      child.on('error', (e: Error) => {
        lifecycle.child = null
        reject(runnerUnavailableError(e.message))
      })
      child.on('close', () => {
        lifecycle.child = null
        if (errState.reason !== null) {
          reject(runnerUnavailableError(errState.reason))
          return
        }
        resolve(messages)
      })
    })
  }

  async function run(args: RunArgs): Promise<TestRunResult> {
    const messages = await driveChild(adapter.buildRunArgs(args, cwd), true)
    // Narrow via guards (isTestEvent → isRunEnd), not Extract<…> predicate-cast filters.
    const runEnd = messages.filter(isTestEvent).filter(isRunEnd).at(-1)
    const result: TestRunResult = runEnd
      ? {summary: runEnd.summary, failures: runEnd.failures, tests: runEnd.tests}
      : cache.last
    cache.last = result
    return result
  }
  async function list(failedOnly = false): Promise<ListResult> {
    const messages = await driveChild(adapter.buildListArgs(failedOnly, cwd), false)
    const listMsg = messages.filter(isListMessage).at(-1)
    return {files: listMsg?.files ?? []}
  }
  function status(): TestRunResult {
    return cache.last
  }
  function emitSnapshot(): TestEvent {
    return {type: 'snapshot', files: [...cache.files.values()], summary: cache.last.summary, watching: false}
  }
  async function openUiServer(): Promise<UiServerInfo> {
    return {available: false}
  }
  function subscribeRaw(cb: (e: TestEvent) => void): () => void {
    subscribers.add(cb)
    for (const e of runBuffer) cb(e)
    return () => subscribers.delete(cb)
  }
  async function stop(): Promise<void> {
    lifecycle.child?.kill('SIGTERM')
    lifecycle.child = null
  }

  return {list, run, status, subscribeRaw, emitSnapshot, openUiServer, stop}
}
```

```ts
  const cases = vitest.state.getTestModules().flatMap((mod) => [...mod.children.allTests()])
  const tests = cases.map(toRow)
  const passed = tests.filter((t) => t.state === 'pass').length
  const failed = tests.filter((t) => t.state === 'fail').length
  const skipped = tests.filter((t) => t.state === 'skip').length
  const durationMs = vitest.state.getTestModules().reduce((sum, mod) => sum + mod.diagnostic().duration, 0)
  const failures = tests.map((t) => t.error).filter((e): e is TestError => e !== undefined)

  return {
    onTestRunStart(specs: ReadonlyArray<{moduleId: string}>): void {
      counter.runs += 1
      send({type: 'run-start', runId: `r${counter.runs}`, files: specs.map((s) => s.moduleId)})
    },
    onTestCaseResult(tc: TestCaseWithDiagnostic): void {

        type: 'test',

    },
    onTestRunEnd(): void {
      const {summary, failures, tests} = collect(getVitest())
      send({type: 'run-end', runId: `r${counter.runs}`, summary, failures, tests})
    },
  }
}
```

```ts
import {createRequire} from 'node:module'
import {pathToFileURL} from 'node:url'
import {join, relative} from 'node:path'

import type {Summary, TestError, TestEvent, TestRow, TestState} from '@opendui/aidx-protocol/test-types'

// OUT-OF-PROCESS jest runner child. Spawned clean by the shared driver; resolves the
// previewed app's own jest (@jest/core runCLI) at runtime and registers a custom reporter
// that maps jest's reporter callbacks to TestEvent NDJSON on fd 3. Jest's native event shape
// is translated here so the protocol stays runner-blind.
type ListFile = {file: string; relPath: string; lastState?: string}
type ChildMessage = TestEvent | {type: 'list'; files: ListFile[]} | {type: 'error'; reason: string}

function send(msg: ChildMessage): void {
  writeSync(3, JSON.stringify(msg) + '\n')
}
function flagValue(argv: string[], name: string): string | undefined {
  const i = argv.indexOf(name)
  return i >= 0 ? argv[i + 1] : undefined
}
function flagValues(argv: string[], name: string): string[] {
  return argv.flatMap((a, i) => {
    const v = argv[i + 1]
    return a === name && v !== undefined ? [v] : []
  })
}
function toState(status: string): TestState {
  if (status === 'passed') return 'pass'
  if (status === 'failed') return 'fail'
  return 'skip'
}

// Jest reporter contract (structural subset of what we read).
type JestAssertion = {
  title: string
  fullName: string
  status: string
  duration?: number | null
  failureMessages: string[]
}
type JestTestFileResult = {
  testFilePath: string
  testResults: JestAssertion[]
  numFailingTests: number
  perfStats?: {runtime?: number}
}
type JestAggregated = {
  numPassedTests: number
  numFailedTests: number
  numPendingTests: number
  testResults: JestTestFileResult[]
}

function firstLine(msg: string): string {
  return (msg.split('\n').find((l) => l.trim().length > 0) ?? msg).trim()
}
function toError(file: string, a: JestAssertion): TestError {
  const message = a.failureMessages[0] ? firstLine(a.failureMessages[0]) : 'test failed'
  return {file, name: a.fullName || a.title, message, stack: a.failureMessages[0] ?? message}
}

// The reporter shape we drive — named so makeReporter is typed at the source and needs no
// cast at the call site.
type JestReporter = {
  onRunStart(): void
  onTestCaseResult(test: {path: string}, tc: JestAssertion): void
  onTestFileResult(test: {path: string}, fileResult: JestTestFileResult): void
  onRunComplete(contexts: unknown, agg: JestAggregated): void
}

function makeReporter(): JestReporter {
  const counter = {runs: 0}
  return {
    onRunStart(): void {
      counter.runs += 1
      send({type: 'run-start', runId: `r${counter.runs}`, files: []})
    },
    onTestCaseResult(test: {path: string}, tc: JestAssertion): void {
      const file = test.path
      send({
        type: 'test',
        file,
        name: tc.fullName || tc.title,
        state: toState(tc.status),
        durationMs: tc.duration ?? 0,
        error: tc.status === 'failed' ? toError(file, tc) : undefined,
      })
    },
    onTestFileResult(_test: {path: string}, fileResult: JestTestFileResult): void {
      send({
        type: 'file-end',
        file: fileResult.testFilePath,
        ok: fileResult.numFailingTests === 0,
        durationMs: fileResult.perfStats?.runtime ?? 0,
      })
    },
    onRunComplete(_contexts: unknown, agg: JestAggregated): void {
      const tests: TestRow[] = agg.testResults.flatMap((f) =>
        f.testResults.map((a) => ({
          file: f.testFilePath,
          name: a.fullName || a.title,
          state: toState(a.status),
          durationMs: a.duration ?? 0,
          error: a.status === 'failed' ? toError(f.testFilePath, a) : undefined,
        })),
      )
      const summary: Summary = {
        passed: agg.numPassedTests,
        failed: agg.numFailedTests,
        skipped: agg.numPendingTests,
        durationMs: tests.reduce((s, t) => s + t.durationMs, 0),
      }
      const failures: TestError[] = tests.map((t) => t.error).filter((e): e is TestError => e !== undefined)
      send({type: 'run-end', runId: `r${counter.runs}`, summary, failures, tests})
    },
  }
}

type JestCoreLike = {
  runCLI: (argv: Record<string, unknown>, projects: string[]) => Promise<{results: JestAggregated}>
}

// Generic runtime-module loader: import to `unknown`, narrow with a guard — replaces
// `(await import(...)) as unknown as JestCoreLike`. (Shared by all runner children; factor
// into a small internal util if you prefer — duplicated inline here so the child is portable.)
function isRecord(v: unknown): v is Record<string, unknown> {
  return typeof v === 'object' && v !== null
}
function hasFn(v: unknown, key: string): boolean {
  return isRecord(v) && key in v && typeof v[key] === 'function'
}
async function loadModule<T>(specifier: string, isModule: (m: unknown) => m is T): Promise<T> {
  const mod: unknown = await import(specifier)
  if (!isModule(mod)) throw new Error(`module "${specifier}" lacks the expected shape`)
  return mod
}
function isJestCore(m: unknown): m is JestCoreLike {
  return hasFn(m, 'runCLI')
}

async function loadJestCore(cwd: string): Promise<JestCoreLike> {
  const req = createRequire(join(cwd, 'noop.js'))
  // Runtime dynamic import — jest is the previewed app's dependency, resolved from its cwd.
  // Narrowed by guard, not asserted.
  return loadModule(pathToFileURL(req.resolve('@jest/core')).href, isJestCore)
}

// `runCLI`'s `results` is typed JestAggregated, which already carries `testResults` — read it
// directly, no `as unknown as {testResults?…}`.
async function runList(cwd: string): Promise<void> {
  const core = await loadJestCore(cwd)
  const cli: Record<string, unknown> = {listTests: true, runInBand: true}
  const {results} = await core.runCLI(cli, [cwd])
  const files = results.testResults ?? []
  send({type: 'list', files: files.map((f) => ({file: f.testFilePath, relPath: relative(cwd, f.testFilePath)}))})
}

async function runTests(cwd: string, argv: string[]): Promise<void> {
  const core = await loadJestCore(cwd)
  const patterns = flagValues(argv, '--pattern')
  const testNamePattern = flagValue(argv, '--name')
  // makeReporter is already typed JestReporter — no cast needed.
  const reporter = makeReporter()
  reporter.onRunStart()
  const cli: Record<string, unknown> = {runInBand: true, _: patterns}
  if (testNamePattern) cli.testNamePattern = testNamePattern
  const {results} = await core.runCLI(cli, [cwd])
  for (const f of results.testResults) {
    for (const a of f.testResults) reporter.onTestCaseResult({path: f.testFilePath}, a)
    reporter.onTestFileResult({path: f.testFilePath}, f)
  }
  reporter.onRunComplete(undefined, results)
}

async function main(): Promise<void> {
  const argv = process.argv.slice(2)
  const mode = flagValue(argv, '--mode') ?? 'run'
  const cwd = flagValue(argv, '--cwd') ?? process.cwd()
  try {
    await (mode === 'list' ? runList(cwd) : runTests(cwd, argv))
    process.exit(0)
  } catch (e) {
    send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
    process.exit(1)
  }
}

main().catch((e: unknown) => {
  send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
  process.exit(1)
})
```

```ts
import {run} from 'node:test'
import {readdir} from 'node:fs/promises'
import {join, relative} from 'node:path'
import {writeSync} from 'node:fs'
import type {Summary, TestError, TestEvent, TestRow, TestState} from '@opendui/aidx-protocol/test-types'

// OUT-OF-PROCESS node:test runner child. node:test is a core module, so no previewed-app
// resolution is needed. run({files}) returns a stream of {type, data} events which we map to
// TestEvent NDJSON on fd 3. The protocol stays runner-blind.
type ListFile = {file: string; relPath: string; lastState?: string}
type ChildMessage = TestEvent | {type: 'list'; files: ListFile[]} | {type: 'error'; reason: string}

function send(msg: ChildMessage): void {
  writeSync(3, JSON.stringify(msg) + '\n')
}
function flagValue(argv: string[], name: string): string | undefined {
  const i = argv.indexOf(name)
  return i >= 0 ? argv[i + 1] : undefined
}

async function findTestFiles(cwd: string): Promise<string[]> {
  const entries = await readdir(cwd, {withFileTypes: true})
  return entries
    .filter((e) => e.isFile() && /\.test\.(js|mjs|cjs|ts)$/.test(e.name))
    .map((e) => join(cwd, e.name))
    .toSorted()
}

type NodeTestEventData = {
  name?: string
  file?: string
  details?: {duration_ms?: number; error?: {message?: string; cause?: {message?: string}}}
}

function errMessage(data: NodeTestEventData): string {
  return data.details?.error?.cause?.message ?? data.details?.error?.message ?? 'test failed'
}

// node:test's run() yields TestEvent objects whose runtime shape we read structurally; a
// guard narrows each streamed value from unknown to {type, data} — no `stream as AsyncIterable`.
type NodeTestStreamEvent = {type: string; data: NodeTestEventData}
function isNodeTestEvent(value: unknown): value is NodeTestStreamEvent {
  return (
    typeof value === 'object' &&
    value !== null &&
    'type' in value &&
    typeof value.type === 'string' &&
    'data' in value &&
    typeof value.data === 'object'
  )
}

async function runTests(cwd: string, files: string[]): Promise<void> {
  send({type: 'run-start', runId: 'r1', files})
  const tests: TestRow[] = []
  const fileFail = new Map<string, boolean>()
  const fileDur = new Map<string, number>()
  const stream: AsyncIterable<unknown> = run({files, concurrency: false})
  for await (const raw of stream) {
    if (!isNodeTestEvent(raw)) continue
    const event = raw
    const data = event.data
    const file = data.file ?? cwd
    const durationMs = data.details?.duration_ms ?? 0
    if (event.type === 'test:pass' || event.type === 'test:fail') {
      const state: TestState = event.type === 'test:pass' ? 'pass' : 'fail'
      const error: TestError | undefined =
        state === 'fail' ? {file, name: data.name ?? '', message: errMessage(data), stack: errMessage(data)} : undefined
      // node:test emits pass/fail for both leaf tests and the file-level suite; we record
      // leaf rows (those with a name) and fold suite results into file aggregation.
      const row: TestRow = {file, name: data.name ?? '', state, durationMs, error}
      tests.push(row)
      send({type: 'test', file, name: row.name, state, durationMs, error})
      if (state === 'fail') fileFail.set(file, true)
      fileDur.set(file, (fileDur.get(file) ?? 0) + durationMs)
    }
  }
  for (const file of files) send({type: 'file-end', file, ok: !fileFail.get(file), durationMs: fileDur.get(file) ?? 0})
  const passed = tests.filter((t) => t.state === 'pass').length
  const failed = tests.filter((t) => t.state === 'fail').length
  const skipped = tests.filter((t) => t.state === 'skip').length
  const summary: Summary = {passed, failed, skipped, durationMs: [...fileDur.values()].reduce((s, d) => s + d, 0)}
  const failures = tests.map((t) => t.error).filter((e): e is TestError => e !== undefined)
  send({type: 'run-end', runId: 'r1', summary, failures, tests})
}

async function main(): Promise<void> {
  const argv = process.argv.slice(2)
  const mode = flagValue(argv, '--mode') ?? 'run'
  const cwd = flagValue(argv, '--cwd') ?? process.cwd()
  try {
    const files = await findTestFiles(cwd)
    if (mode === 'list') {
      send({type: 'list', files: files.map((f) => ({file: f, relPath: relative(cwd, f)}))})
    } else {
      await runTests(cwd, files)
    }
    process.exit(0)
  } catch (e) {
    send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
    process.exit(1)
  }
}

main().catch((e: unknown) => {
  send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
  process.exit(1)
})
```

```ts
// OUT-OF-PROCESS playwright runner child. THE LOOSER-FIT CASE: Playwright is e2e with a real
// browser lifecycle, so its model doesn't map 1:1 onto a unit-test runner. We run Playwright
// with its JSON reporter, then translate the JSON report onto the SAME TestEvent contract.
// CAVEAT: there's no per-test streaming here (we read the final JSON), so live `test` events
// are synthesized from the report after the run completes — the card still rebuilds the full
// tree, but mid-run streaming granularity is coarser than vitest's. capabilities.watch=false.
type ListFile = {file: string; relPath: string; lastState?: string}
type ChildMessage = TestEvent | {type: 'list'; files: ListFile[]} | {type: 'error'; reason: string}

function send(msg: ChildMessage): void {
  writeSync(3, JSON.stringify(msg) + '\n')
}
function flagValue(argv: string[], name: string): string | undefined {
  const i = argv.indexOf(name)
  return i >= 0 ? argv[i + 1] : undefined
}

type PwResult = {status: string; duration?: number; error?: {message?: string}; errors?: {message?: string}[]}
type PwTest = {results: PwResult[]}
type PwSpec = {title: string; file?: string; tests: PwTest[]}
type PwSuite = {file?: string; title?: string; specs?: PwSpec[]; suites?: PwSuite[]}
type PwReport = {
  suites: PwSuite[]
  stats?: {expected?: number; unexpected?: number; skipped?: number; duration?: number}
}

function toState(status: string): TestState {
  if (status === 'passed' || status === 'expected') return 'pass'
  if (status === 'skipped') return 'skip'
  return 'fail'
}

// Flatten Playwright's nested suites into {file, name, result} rows.
function flatten(suite: PwSuite, file: string, rows: {file: string; name: string; result: PwResult}[]): void {
  const here = suite.file ?? file
  for (const spec of suite.specs ?? []) {
    const last = spec.tests[0]?.results.at(-1)
    if (last) rows.push({file: spec.file ?? here, name: spec.title, result: last})
  }
  for (const child of suite.suites ?? []) flatten(child, here, rows)
}

function strip(msg: string): string {
  // Playwright error messages carry ANSI codes; keep the first non-empty line, de-ansied.
  // eslint-disable-next-line no-control-regex
  const clean = msg.replace(/\[[0-9;]*m/g, '')
  return (clean.split('\n').find((l) => l.trim().length > 0) ?? clean).trim()
}

// Parse the JSON report to `unknown`, then narrow with a guard — never `JSON.parse(...) as
// PwReport`. We only require the `suites` array to be present; everything below it is read
// through optional fields, so a shallow guard is enough to trust the shape.
function isPwReport(value: unknown): value is PwReport {
  return typeof value === 'object' && value !== null && 'suites' in value && Array.isArray(value.suites)
}

async function runReport(cwd: string): Promise<PwReport> {
  const dir = await mkdtemp(join(tmpdir(), 'aidx-pw-'))
  const out = join(dir, 'report.json')
  const req = createRequire(join(cwd, 'noop.js'))
  // Runtime resolution of the previewed app's @playwright/test CLI. We invoke its programmatic
  // runner with the json reporter targeted at `out`. The import is for side-effect (resolving
  // + loading the package); its result is unused, so it is not typed/asserted.
  await import(pathToFileURL(req.resolve('@playwright/test')).href)
  const {spawn} = await import('node:child_process')
  const cliBin = req.resolve('@playwright/test/cli')
  await new Promise<void>((resolve, reject) => {
    const proc = spawn(process.execPath, [cliBin, 'test', '--reporter=json'], {
      cwd,
      env: {...process.env, PLAYWRIGHT_JSON_OUTPUT_NAME: out},
      stdio: 'ignore',
    })
    proc.on('error', reject)
    proc.on('close', () => resolve()) // non-zero exit is expected when tests fail
  })
  const parsed: unknown = JSON.parse(await readFile(out, 'utf8'))
  if (!isPwReport(parsed)) throw new Error('playwright JSON report missing a suites array')
  return parsed
}

async function runTests(cwd: string): Promise<void> {
  send({type: 'run-start', runId: 'r1', files: []})
  const report = await runReport(cwd)
  const rows: {file: string; name: string; result: PwResult}[] = []
  for (const suite of report.suites) flatten(suite, cwd, rows)
  const tests: TestRow[] = rows.map((r) => {
    const state = toState(r.result.status)
    const msg = r.result.error?.message ?? r.result.errors?.[0]?.message
    const error: TestError | undefined =
      state === 'fail'
        ? {file: r.file, name: r.name, message: msg ? strip(msg) : 'test failed', stack: msg ?? ''}
        : undefined
    return {file: r.file, name: r.name, state, durationMs: r.result.duration ?? 0, error}
  })
  const byFile = new Map<string, {ok: boolean; dur: number}>()
  for (const t of tests) {
    const cur = byFile.get(t.file) ?? {ok: true, dur: 0}
    byFile.set(t.file, {ok: cur.ok && t.state !== 'fail', dur: cur.dur + t.durationMs})
    send({type: 'test', file: t.file, name: t.name, state: t.state, durationMs: t.durationMs, error: t.error})
  }
  for (const [file, agg] of byFile) send({type: 'file-end', file, ok: agg.ok, durationMs: agg.dur})
  const summary: Summary = {
    passed: tests.filter((t) => t.state === 'pass').length,
    failed: tests.filter((t) => t.state === 'fail').length,
    skipped: tests.filter((t) => t.state === 'skip').length,
    durationMs: report.stats?.duration ?? tests.reduce((s, t) => s + t.durationMs, 0),
  }
  const failures = tests.map((t) => t.error).filter((e): e is TestError => e !== undefined)
  send({type: 'run-end', runId: 'r1', summary, failures, tests})
}

async function main(): Promise<void> {
  const argv = process.argv.slice(2)
  const mode = flagValue(argv, '--mode') ?? 'run'
  const cwd = flagValue(argv, '--cwd') ?? process.cwd()
  try {
    if (mode === 'list') {
      // Playwright has no cheap file-list without a discovery run; report empty (the card
      // tolerates an empty list and the run populates the tree). Documented coarse fit.
      send({type: 'list', files: []})
    } else {
      await runTests(cwd)
    }
    process.exit(0)
  } catch (e) {
    send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
    process.exit(1)
  }
}

main().catch((e: unknown) => {
  send({type: 'error', reason: e instanceof Error ? e.message : String(e)})
  process.exit(1)
})
```
