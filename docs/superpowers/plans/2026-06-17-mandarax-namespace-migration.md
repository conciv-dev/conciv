# @mandarax Namespace Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move every package from the `@mandarax/*` scope to `@mandarax/*`, add a single publishable umbrella package `mandarax` that re-exports the plugin, and leave **zero** `mandarax` references anywhere in the repo.

**Architecture:** Pure rename via a single string transform (`@mandarax/` -> `@mandarax/`) across all source, configs, package manifests, and docs, plus one targeted fix for the bare regex form (`/^@mandarax\/mandarax-/`). A new `packages/mandarax` workspace package (name `mandarax`) ships thin re-export files that forward `mandarax/plugin/*` to `@mandarax/plugin/*`, with the plugin kept as an external dependency (not bundled). Example apps switch to consuming the umbrella.

**Tech Stack:** pnpm workspaces, turborepo, tsdown (build), tsc (typecheck), vitest + Playwright (tests).

## Global Constraints

- Folder names DO NOT change. Only package `name`, `exports`, `dependencies`, import specifiers, and doc text change. (`packages/plugin` stays `packages/plugin`; its name becomes `@mandarax/plugin`.)
- The transform is exactly: `@mandarax/` -> `@mandarax/`. Example: `@mandarax/protocol` -> `@mandarax/protocol`, `@mandarax/plugin/vite` -> `@mandarax/plugin/vite`.
- The bare regex form `/^@mandarax\/mandarax-/` is NOT matched by the slash transform (a `\` sits between `mandarax` and `/`); it must be fixed separately to `/^@mandarax\/mandarax-/`.
- The word "mandarax" as a product/name (bundle filename `mandarax-widget.global.js`, bin name, env vars like `NEXT_PUBLIC_MANDARAX_PORT`, dirs) is NOT in scope. Only the `mandarax` npm scope changes.
- Do NOT hand-edit `pnpm-lock.yaml`; it is regenerated by `pnpm install`.
- Acceptance gate (the literal definition of done): `grep -rn 'mandarax' . --exclude-dir=node_modules --exclude-dir=.git` returns nothing, after a clean rebuild.
- The 9 existing packages and their final names:
  - `@mandarax/cli` -> `@mandarax/cli`
  - `@mandarax/core` -> `@mandarax/core`
  - `@mandarax/harness` -> `@mandarax/harness`
  - `@mandarax/plugin` -> `@mandarax/plugin`
  - `@mandarax/protocol` -> `@mandarax/protocol`
  - `@mandarax/solid-streamdown` -> `@mandarax/solid-streamdown`
  - `@mandarax/test-runner` -> `@mandarax/test-runner`
  - `@mandarax/tools` -> `@mandarax/tools`
  - `@mandarax/widget` -> `@mandarax/widget`
- New package: `mandarax` in folder `packages/mandarax`.
