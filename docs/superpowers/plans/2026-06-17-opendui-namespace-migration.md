# @opendui Namespace Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move every package from the `@opendui/aidx-*` scope to `@opendui/aidx-*`, add a single publishable umbrella package `@opendui/aidx` that re-exports the plugin, and leave **zero** `@opendui/aidx` references anywhere in the repo.

**Architecture:** Pure rename via a single string transform (`@opendui/aidx-` -> `@opendui/aidx-`) across all source, configs, package manifests, and docs, plus one targeted fix for the bare regex form (`/^@opendui\/aidx-/`). A new `packages/aidx` workspace package (name `@opendui/aidx`) ships thin re-export files that forward `@opendui/aidx/plugin/*` to `@opendui/aidx-plugin/*`, with the plugin kept as an external dependency (not bundled). Example apps switch to consuming the umbrella.

**Tech Stack:** pnpm workspaces, turborepo, tsdown (build), tsc (typecheck), vitest + Playwright (tests).

## Global Constraints

- Folder names DO NOT change. Only package `name`, `exports`, `dependencies`, import specifiers, and doc text change. (`packages/plugin` stays `packages/plugin`; its name becomes `@opendui/aidx-plugin`.)
- The transform is exactly: `@opendui/aidx-` -> `@opendui/aidx-`. Example: `@opendui/aidx-protocol` -> `@opendui/aidx-protocol`, `@opendui/aidx-plugin/vite` -> `@opendui/aidx-plugin/vite`.
- The bare regex form `/^@opendui\/aidx-/` is NOT matched by the slash transform (a `\` sits between `@opendui/aidx` and `/`); it must be fixed separately to `/^@opendui\/aidx-/`.
- The word "aidx" as a product/name (bundle filename `aidx-widget.global.js`, bin name, env vars like `NEXT_PUBLIC_AIDX_PORT`, dirs) is NOT in scope. Only the `@opendui/aidx` npm scope changes.
- Do NOT hand-edit `pnpm-lock.yaml`; it is regenerated by `pnpm install`.
- Acceptance gate (the literal definition of done): `grep -rn '@opendui/aidx' . --exclude-dir=node_modules --exclude-dir=.git` returns nothing, after a clean rebuild.
- The 9 existing packages and their final names:
  - `@opendui/aidx-cli` -> `@opendui/aidx-cli`
  - `@opendui/aidx-core` -> `@opendui/aidx-core`
  - `@opendui/aidx-harness` -> `@opendui/aidx-harness`
  - `@opendui/aidx-plugin` -> `@opendui/aidx-plugin`
  - `@opendui/aidx-protocol` -> `@opendui/aidx-protocol`
  - `@opendui/aidx-solid-streamdown` -> `@opendui/aidx-solid-streamdown`
  - `@opendui/aidx-test-runner` -> `@opendui/aidx-test-runner`
  - `@opendui/aidx-tools` -> `@opendui/aidx-tools`
  - `@opendui/aidx-widget` -> `@opendui/aidx-widget`
- New package: `@opendui/aidx` in folder `packages/aidx`.
