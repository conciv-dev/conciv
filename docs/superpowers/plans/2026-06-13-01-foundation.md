# aidx Adapter Architecture — Plan 1: Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **HARD RULES (see the spec's "Coding conventions" — non-negotiable):** No casts (`as` except `as const`), no `!`, no IIFEs, no `index.ts`. **Zod for all parsed data** — h3 `readValidatedBody`/`getValidatedQuery` for HTTP bodies/query, `Schema.safeParse(JSON.parse(...))` for NDJSON/stream-json/JSON files; NEVER hand-roll `isRecord`/`typeof` data guards. Every interface ships a generic `defineX<T extends X>`; author implementations through it (never a bare literal). HTTP routes under `api/` (paths `/api/...`), one naming convention (entry file named after its folder, no `-route`/`-gate`). `@opendui/aidx-core` imports no bundler — dev-server access via `BundlerBridge`. Pin **h3 2.0.1-rc.22** (2.0.0 broken); use `event.req` (web Request), `event.res.status = N`, return objects→JSON, `new Response(stream, {headers})` for SSE; srvx `serve({fetch: app.fetch, port})`. Terse comments only. Commit to `main`. **Verify library APIs against real online docs, not `.d.ts` grepping.**

**Goal:** Lay the foundation for the adapter architecture by (1) generalizing `@opendui/aidx-protocol` types so no type name mentions "vitest", adding `harness-types.ts` and `runner-types.ts`; (2) extracting the framework-free engine into a new `@opendui/aidx-core` package built on h3 v2 + srvx, wiring claude + vitest through registry seams that the dedicated harness/runner packages will replace in Plans 2 & 3; and (3) keeping `@opendui/aidx-vite-plugin` working as a thin consumer of `@opendui/aidx-core`. Every task ends green.

**Architecture:** Three orthogonal seams (harness, test-runner, bundler) layered on a standalone h3 engine. This plan is the riskiest because Plans 2–4 depend on it. `@opendui/aidx-protocol` is types plus the tiny `define*` factory runtime (`defineHarness`/`defineRunner`/`defineConfig` — generic identity helpers that lock types + dev-assert capability invariants; everything else is zero-runtime types). `@opendui/aidx-core` is a framework-free h3 app exposing `start(opts) => {port, stop}` and `htmlTags(corePort, opts)`; it boots its own srvx server on its own port (cross-origin, so `pw-api-base` meta points at the core port). `@opendui/aidx-vite-plugin` becomes a thin Vite `Plugin` whose `configureServer` boots `core.start()` and whose `transformIndexHtml` injects `core.htmlTags()`. Claude and vitest remain the only adapters, held inline inside core behind a `harnessRegistry`/`runnerRegistry` seam (the dedicated `@opendui/aidx-harness`/`@opendui/aidx-runner` packages land later).

**Tech Stack:** ESM, Node 22, TypeScript with `verbatimModuleSyntax`. **Functions not classes. No IIFEs. No `index.ts` barrels** (named files only; package `exports` point at named files). Build = tsdown, test = `vitest run` per package, typecheck = `tsc --noEmit`, lint = oxlint, format = oxfmt. h3 is v2 (web-standard `Request`/`Response`/`ReadableStream`); listener via srvx `serve({fetch, port})`. Commits go straight to `main` (no branches).

```ts
// Wire types streamed over /api/test-runner/stream and returned by the run/status/list routes.
// Mirrored verbatim by the widget's test card — never guessed there. `parseFailure` is the
// one pure bit worth unit-testing: it maps a Vitest v4 TestCase (shape below) to our flat
// TestError. NO runner name appears in any type below — these are the runner-blind wrapper
// contract every runner adapter translates its native output into.

export type TestState = 'pass' | 'fail' | 'skip'

export type TestError = {file: string; name: string; message: string; stack: string; line?: number}

export type Summary = {passed: number; failed: number; skipped: number; durationMs: number}

export type FileState = {file: string; state: TestState | 'running'; durationMs?: number}

// One completed test. `run-end` carries the FULL list (every pass/fail/skip) so a card
// rendered from a stored tool-result on reload can rebuild the whole tree — not just the
// failures. `error` is present only for failures.
export type TestRow = {file: string; name: string; state: TestState; durationMs: number; error?: TestError}

// The result returned by `aidx tools test run` (printed as the tool-result the widget
// renders as a card) and by the run route. Carries the full tree so it's self-contained.
export type TestRunResult = {summary: Summary; failures: TestError[]; tests: TestRow[]}

export type TestEvent =
  | {type: 'snapshot'; files: FileState[]; summary: Summary; watching: boolean}
  | {type: 'run-start'; runId: string; files: string[]}
  | {type: 'test'; file: string; name: string; state: TestState; durationMs: number; error?: TestError}
  | {type: 'file-end'; file: string; ok: boolean; durationMs: number}
  | {type: 'run-end'; runId: string; summary: Summary; failures: TestError[]; tests: TestRow[]}

// Structural subset of a Vitest v4 TestCase — only what parseFailure reads.
export type TestCaseLike = {
  name: string
  module: {moduleId: string}
  result: () => {
    state: 'passed' | 'failed' | 'skipped' | 'pending'
    errors?: ReadonlyArray<{message: string; stacks?: ReadonlyArray<{file?: string; line?: number}>}>
  }
}

export function parseFailure(tc: TestCaseLike): TestError | null {
  const result = tc.result()
  if (result.state !== 'failed') return null
  const err = result.errors?.[0]
  const message = err?.message ?? 'test failed'
  const top = err?.stacks?.find((s) => s.file === tc.module.moduleId) ?? err?.stacks?.[0]
  return {
    file: tc.module.moduleId,
    name: tc.name,
    message,
    stack: message,
    line: top?.line,
  }
}
```

```ts
// Core resolves a HarnessAdapter and feature-detects by capability, degrading gracefully
// when a harness lacks resume / a permission gate / transcript history / a system-prompt flag.

export type HarnessCapabilities = {
  resume: boolean // can --resume a prior session
  permissionGate: 'hook' | 'none' // can call back mid-turn for tool approval
  transcriptHistory: boolean // can hydrate prior turns from disk
  systemPrompt: 'file' | 'flag' | 'none'

  cwd: string
  resumeSessionId: string | null
  systemPrompt: string
  permissionUrl?: string // provided by core iff permissionGate === 'hook'
}

export type HarnessChild = {pid: number; stdout: Readable; stderr: Readable; kill(): void}

  id: string // 'claude' | 'codex' | …
  binName: string // default binary on PATH
  capabilities: HarnessCapabilities
  buildArgs(turn: HarnessTurn): string[]
  decode(lines: AsyncIterable<string>, opts: {onSessionId(id: string): void}): AsyncGenerator<StreamChunk>
  transcriptPath?(cwd: string, sessionId: string): string // present iff transcriptHistory
  parseHistory?(raw: string): UIMessage[] // present iff transcriptHistory
}
```

```ts
  options: AidxConfig
  root: string
  bridge?: BundlerBridge // the live dev server, abstracted; absent for build-only bundlers
  launchEditor: (file: string, line: number) => void
  // The bundler may inject extra env (e.g. PATH with the aidx CLI shim, AIDX_PORT).
  childEnv?: (corePort: number) => NodeJS.ProcessEnv

export async function start(opts: StartOpts): Promise<Engine> {
  const cfg = resolveConfig(opts.options, opts.root)
  const stateDir = join(cfg.lockDir, '.aidx')
  mkdirSync(stateDir, {recursive: true})
  const systemPromptFile = join(stateDir, 'chat-system-prompt.txt')
  writeFileSync(systemPromptFile, cfg.systemPrompt)

  const openInEditor = makeEditorOpener(
    (file, line) => opts.launchEditor(file, line),
    4000,
    () => Date.now(),
  )

  // The spawn seam: bundler-provided env (PATH shim, AIDX_PORT) layered over a default spawn.
  const portRef = {port: 0}
  const spawnHarness = (args: string[], cwd: string): HarnessChild => {
    const harnessBin = cfg.harnessBin ?? 'claude'
    const env = opts.childEnv ? opts.childEnv(portRef.port) : process.env
    const child = spawn(harnessBin, args, {cwd, stdio: ['ignore', 'pipe', 'pipe'], env})
    // No non-null assertions (`child.stdout!`): `stdio:[…'pipe','pipe']` guarantees the pipes,
    // but assert it through a guard so the types narrow without `!` (spec: never assertions).
    const {stdout, stderr} = child
    if (!stdout || !stderr) throw new Error(`harness "${harnessBin}" did not expose stdout/stderr pipes`)
    return {pid: child.pid ?? -1, stdout, stderr, kill: () => void child.kill('SIGTERM')}
  }

    cfg,
    cwd: opts.root,
    bridge: opts.bridge,
    openInEditor,
    systemPromptFile,
    spawnHarness,
  }
  const app = makeApp(appOpts)
  const server = serve({fetch: app.fetch, port: 0})
  await server.ready()
  portRef.port = server.port
  return {
    port: server.port,
    cfg,
    stop: async () => {
      await server.close()
    },
  }
}
```

```ts
const require = createRequire(import.meta.url)

function resolveWidgetFile(): string | null {
  try {
    return require.resolve('@opendui/aidx-widget/global')
  } catch {
    return null
  }
}

// The Vite BundlerBridge: wraps the live ViteDevServer's accessors (tools-layer.ts) into the
// bundler-agnostic interface core consumes. Authored through defineBundlerBridge (never a bare
// object literal). reload/restart drive HMR off the live module graph.
function makeViteBridge(server: ViteLike): BundlerBridge {
  return defineBundlerBridge({
    id: 'vite',
    config: () => viteConfig(server),
    resolve: (spec, importer) => viteResolve(server, spec, importer),
    moduleGraph: (file) => viteGraph(server, file),
    transform: (url) => viteTransform(server, url),
    urls: () => viteUrls(server),
    reload: async (file) => {
      const mods = server.moduleGraph.getModulesByFile(file)
      if (mods && server.reloadModule) for (const m of mods) await server.reloadModule(m)
    },
    restart: async (force) => {
      await server.restart?.(force)
    },
  })
}

export function aidx(options: AidxConfig = {}): Plugin {
  const widgetFile = resolveWidgetFile()
  const effectiveWidgetUrl = options.widgetUrl ?? (widgetFile ? DEFAULT_WIDGET_ROUTE : undefined)
  const serveBundledWidget = !options.widgetUrl && widgetFile !== null
  const engineRef: {engine: Engine | null} = {engine: null} // annotated binding, not `as`
  return {
    name: 'aidx',
    apply: 'serve',
    transformIndexHtml: {
      order: 'pre',
      handler(_html, ctx) {
        const cfg = resolveConfig(options, ctx.server?.config.root ?? process.cwd())
        if (!cfg.enabled || !engineRef.engine) return []
        return htmlTags(engineRef.engine.port, {previewId: cfg.previewId, widgetUrl: effectiveWidgetUrl})
      },
    },
    async configureServer(server) {
      const root = server.config.root
      const cfg = resolveConfig(options, root)
      if (!cfg.enabled) return

      if (effectiveWidgetUrl) {
        server.middlewares.stack.unshift({route: '', handle: makeWidgetInject(effectiveWidgetUrl, cfg.previewId)})
      }
      if (serveBundledWidget && widgetFile) server.middlewares.use(makeWidgetServe(widgetFile))

      // CLI shim on the spawned agent's PATH so `aidx tools` resolves to @opendui/aidx-cli.
      const binDir = join(cfg.lockDir, '.aidx', 'bin')
      mkdirSync(binDir, {recursive: true})
      const shim = join(binDir, 'aidx')
      try {
        const binPath = require.resolve('@opendui/aidx-cli/bin')
        rmSync(shim, {force: true})
        symlinkSync(binPath, shim)
      } catch {
        // best effort — fall back to whatever `aidx` is on PATH.
      }
      const agentPath = `${binDir}${delimiter}${process.env.PATH ?? ''}`

      const engine = await start({
        options,
        root,
        bridge: makeViteBridge(server), // BundlerBridge over the live dev server — core imports no vite
        launchEditor: (file, line) => launchEditor(`${file}:${line}`),
        childEnv: (corePort) => ({...process.env, PATH: agentPath, AIDX_PORT: String(corePort)}),
      })
      engineRef.engine = engine
      server.httpServer?.on('close', () => void engine.stop())
    },
  }
}
```
