# @conciv Namespace Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move every package from the `@conciv/*` scope to `@conciv/*`, add a single publishable umbrella package `conciv` that re-exports the plugin, and leave **zero** `conciv` references anywhere in the repo.

**Architecture:** Pure rename via a single string transform (`@conciv/` -> `@conciv/`) across all source, configs, package manifests, and docs, plus one targeted fix for the bare regex form (`/^@conciv\/conciv-/`). A new `packages/conciv` workspace package (name `conciv`) ships thin re-export files that forward `conciv/plugin/*` to `@conciv/plugin/*`, with the plugin kept as an external dependency (not bundled). Example apps switch to consuming the umbrella.

**Tech Stack:** pnpm workspaces, turborepo, tsdown (build), tsc (typecheck), vitest + Playwright (tests).

## Global Constraints

- Folder names DO NOT change. Only package `name`, `exports`, `dependencies`, import specifiers, and doc text change. (`packages/plugin` stays `packages/plugin`; its name becomes `@conciv/plugin`.)
- The transform is exactly: `@conciv/` -> `@conciv/`. Example: `@conciv/protocol` -> `@conciv/protocol`, `@conciv/plugin/vite` -> `@conciv/plugin/vite`.
- The bare regex form `/^@conciv\/conciv-/` is NOT matched by the slash transform (a `\` sits between `conciv` and `/`); it must be fixed separately to `/^@conciv\/conciv-/`.
- The word "conciv" as a product/name (bundle filename `conciv-widget.global.js`, bin name, env vars like `NEXT_PUBLIC_CONCIV_PORT`, dirs) is NOT in scope. Only the `conciv` npm scope changes.
- Do NOT hand-edit `pnpm-lock.yaml`; it is regenerated by `pnpm install`.
- Acceptance gate (the literal definition of done): `grep -rn 'conciv' . --exclude-dir=node_modules --exclude-dir=.git` returns nothing, after a clean rebuild.
- The 9 existing packages and their final names:
  - `@conciv/cli` -> `@conciv/cli`
  - `@conciv/core` -> `@conciv/core`
  - `@conciv/harness` -> `@conciv/harness`
  - `@conciv/plugin` -> `@conciv/plugin`
  - `@conciv/protocol` -> `@conciv/protocol`
  - `@conciv/solid-streamdown` -> `@conciv/solid-streamdown`
  - `@conciv/test-runner` -> `@conciv/test-runner`
  - `@conciv/tools` -> `@conciv/tools`
  - `@conciv/widget` -> `@conciv/widget`
- New package: `conciv` in folder `packages/conciv`.

## Open Risks

- `export {default} from '@conciv/plugin/vite'` requires the plugin's vite entry to keep its default export (it does today: `export default unplugin.vite`). If a future plugin refactor drops the default, the umbrella re-export breaks — Task 3 Step 6 smoke test catches this.
